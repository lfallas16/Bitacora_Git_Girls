Requerimientos Funcionales

  • ☆ Los usuarios deben poder iniciar sesión con credenciales (usuario y contraseña).
  • ☆ El sistema mostrará un mensaje de error en caso de credenciales incorrectas.
  • ☆ El sistema validará credenciales para evitar accesos no autorizados.
  • ☆ El sistema debe identificar el rol del usuario entre administrador, maestro y padre de familia, asignando permisos específicos.
  • ☆ Una vez autenticados, los usuarios accederán a un panel de control con opciones según su rol dentro del sistema.
  • ☆ El usuario administrador tendrá control total sobre la plataforma.
  • ☆ El usuario administrador podrá registrar, modificar, eliminar o visualizar los niveles educativos.
  • ☆ El usuario administrador podrá administrar los materiales escolares.
  • ☆ El usuario administrador podrá administrar los usuarios (registrar, modificar, desactivar cuentas, generar reportes sobre el uso del sistema).
  • ☆ El usuario maestro podrá crear y gestionar listas de útiles escolares.
  • ☆ El usuario maestro podrá seleccionar materiales previamente registrados, organizar los materiales y actualizar o eliminar listas existentes.
  • ☆ El usuario padre de familia podrá solamente visualizar las listas de útiles escolares asociadas al nivel educativo de sus hijos.
  • ☆ El sistema contará con un módulo de gestión de categorías de materiales (papelería, herramientas artísticas, insumos tecnológicos.)
  • ☆ El sistema incluirá una función para la generación de reportes detallados sobre las listas creadas.
  • ☆ El sistema establecerá controles de acceso para garantizar que cada usuario solo pueda acceder la información correspondiente a su rol.
  • ☆ El sistema será escalable para permitir su futura integración con otras plataformas del Ministerio de Educación Pública.
